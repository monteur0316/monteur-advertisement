# Feature Specification: 자주묻는 질문 (FAQ) 페이지

**Feature Branch**: `005-faq-page`
**Created**: 2026-03-03
**Status**: Draft
**Input**: User description: "자주묻는 질문 페이지를 만들고 싶어 해당 페이지에서는 어드민은 CRUD 를 할 수 있어야하고 그 외에는 읽을 수만 있게 해주면 돼"

## Clarifications

### Session 2026-03-03

- Q: FAQ 페이지의 네비게이션 위치는? → A: 조직 사이드바 "일반" 그룹에 FAQ 메뉴 추가. 슈퍼 어드민은 같은 페이지(`/org/[slug]/faq`)에서 CRUD 컨트롤이 추가로 표시됨.
- Q: 표시 순서 지정 방식은? → A: 드래그앤드롭으로 FAQ 목록에서 항목을 드래그하여 순서 변경.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - FAQ 목록 조회 (Priority: P1)

인증된 모든 사용자가 조직 사이드바의 "일반" 그룹에 있는 FAQ 메뉴를 통해 자주묻는 질문 목록을 조회할 수 있다. 질문 항목을 클릭하면 답변이 펼쳐지는 아코디언 형태로 표시된다. 슈퍼 어드민은 같은 페이지에서 CRUD 컨트롤이 추가로 표시된다.

**Why this priority**: FAQ 페이지의 핵심 기능이며 모든 사용자에게 가장 기본적인 가치를 제공한다. 이 기능만으로도 독립적으로 사용 가능한 MVP이다.

**Independent Test**: FAQ 목록 페이지에 접근하여 등록된 질문들이 표시되고, 질문을 클릭하면 답변이 펼쳐지는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 인증된 사용자가 시스템에 로그인한 상태, **When** FAQ 페이지에 접근하면, **Then** 공개된 모든 FAQ 항목이 정렬된 순서로 표시된다
2. **Given** FAQ 목록이 표시된 상태, **When** 질문 항목을 클릭하면, **Then** 해당 질문의 답변이 아코디언 형태로 펼쳐진다
3. **Given** FAQ 항목이 없는 상태, **When** FAQ 페이지에 접근하면, **Then** "등록된 질문이 없습니다" 안내 메시지가 표시된다

---

### User Story 2 - FAQ 항목 등록 (Priority: P2)

슈퍼 어드민이 새로운 FAQ 항목을 등록할 수 있다. 질문과 답변을 입력하여 저장하며, 새 항목은 목록 맨 아래에 추가된다.

**Why this priority**: FAQ 콘텐츠가 존재해야 조회 기능이 의미를 가진다. 관리자가 질문/답변을 생성하는 것은 FAQ 시스템 운영의 핵심이다.

**Independent Test**: 슈퍼 어드민으로 로그인 후 새 FAQ 항목을 작성하고 저장한 뒤, 목록에 정상적으로 나타나는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 슈퍼 어드민이 FAQ 관리 페이지에 있을 때, **When** "새 질문 추가" 버튼을 클릭하면, **Then** 질문과 답변을 입력할 수 있는 폼이 표시된다
2. **Given** 질문과 답변을 입력한 상태, **When** 저장 버튼을 클릭하면, **Then** FAQ 항목이 저장되고 목록에 반영된다
3. **Given** 필수 입력값(질문 또는 답변)이 비어있는 상태, **When** 저장을 시도하면, **Then** 적절한 유효성 검증 오류 메시지가 표시된다

---

### User Story 3 - FAQ 항목 수정 (Priority: P3)

슈퍼 어드민이 기존 FAQ 항목의 질문과 답변을 수정할 수 있다. 표시 순서는 드래그앤드롭으로 별도 변경한다.

**Why this priority**: 등록된 FAQ 내용의 오류 수정이나 내용 업데이트는 콘텐츠 관리에 필수적이다.

**Independent Test**: 슈퍼 어드민으로 로그인 후 기존 FAQ 항목을 수정하고 변경사항이 반영되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 슈퍼 어드민이 FAQ 목록을 보고 있을 때, **When** FAQ 항목의 수정 버튼을 클릭하면, **Then** 기존 내용이 채워진 수정 폼이 표시된다
2. **Given** 수정 폼에서 내용을 변경한 상태, **When** 저장 버튼을 클릭하면, **Then** 변경사항이 저장되고 목록에 반영된다

---

### User Story 4 - FAQ 항목 삭제 (Priority: P4)

슈퍼 어드민이 불필요한 FAQ 항목을 삭제할 수 있다. 삭제 전 확인 절차가 포함된다.

**Why this priority**: 더 이상 유효하지 않은 질문을 정리하는 기능은 콘텐츠 관리에 필요하지만, 등록/수정보다 우선순위가 낮다.

**Independent Test**: 슈퍼 어드민으로 로그인 후 FAQ 항목 삭제를 시도하고 확인 후 삭제되는지 검증한다.

**Acceptance Scenarios**:

1. **Given** 슈퍼 어드민이 FAQ 목록을 보고 있을 때, **When** FAQ 항목의 삭제 버튼을 클릭하면, **Then** 삭제 확인 대화상자가 표시된다
2. **Given** 삭제 확인 대화상자에서, **When** 삭제를 확인하면, **Then** FAQ 항목이 삭제되고 목록에서 제거된다
3. **Given** 삭제 확인 대화상자에서, **When** 취소를 선택하면, **Then** FAQ 항목이 유지된다

---

### Edge Cases

- 슈퍼 어드민이 아닌 사용자가 FAQ 관리 기능(생성/수정/삭제)에 직접 접근을 시도하면 어떻게 되는가? → 접근 거부 처리
- 두 명의 슈퍼 어드민이 동시에 같은 FAQ 항목을 수정하면 어떻게 되는가? → 마지막 저장이 반영됨 (last-write-wins)
- FAQ 항목이 매우 많을 때 (100개 이상) 목록 표시는 어떻게 되는가? → 페이지네이션 없이 전체 표시 (FAQ 특성상 항목 수가 제한적)
- 질문이나 답변에 매우 긴 텍스트가 입력되면 어떻게 되는가? → 질문 최대 200자, 답변 최대 2000자 제한

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 조직 사이드바 "일반" 그룹에 FAQ 메뉴를 제공하고, 인증된 모든 사용자에게 FAQ 목록을 표시해야 한다
- **FR-002**: FAQ 항목은 질문과 답변으로 구성되며, 지정된 순서로 표시되어야 한다
- **FR-003**: 사용자는 질문을 클릭하여 답변을 펼치거나 접을 수 있어야 한다
- **FR-004**: 슈퍼 어드민만 FAQ 항목을 생성할 수 있어야 한다
- **FR-005**: 슈퍼 어드민만 기존 FAQ 항목을 수정할 수 있어야 한다
- **FR-006**: 슈퍼 어드민만 FAQ 항목을 삭제할 수 있어야 한다
- **FR-007**: FAQ 삭제 시 사용자에게 확인을 요청해야 한다
- **FR-008**: 질문은 필수 입력이며 최대 200자로 제한해야 한다
- **FR-009**: 답변은 필수 입력이며 최대 2000자로 제한해야 한다
- **FR-010**: 슈퍼 어드민이 아닌 사용자에게는 생성/수정/삭제 버튼이 표시되지 않아야 한다
- **FR-011**: 슈퍼 어드민이 아닌 사용자의 생성/수정/삭제 요청은 서버 측에서도 거부되어야 한다
- **FR-012**: 슈퍼 어드민은 FAQ 목록에서 드래그앤드롭으로 항목의 표시 순서를 변경할 수 있어야 한다
- **FR-013**: FAQ 항목이 없을 때 빈 상태 안내 메시지를 표시해야 한다

### Key Entities

- **FAQ 항목**: 자주묻는 질문 하나의 단위. 질문(제목), 답변(본문), 표시 순서, 생성 일시, 수정 일시로 구성

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 인증된 사용자가 FAQ 페이지에 접근 후 3초 이내에 전체 질문 목록이 표시된다
- **SC-002**: 슈퍼 어드민이 새 FAQ 항목을 1분 이내에 작성하고 저장할 수 있다
- **SC-003**: 슈퍼 어드민이 FAQ 항목을 수정하고 30초 이내에 저장할 수 있다
- **SC-004**: 슈퍼 어드민이 아닌 사용자에게는 관리 기능이 100% 숨겨진다
- **SC-005**: 비인가 사용자의 관리 작업 요청은 100% 서버 측에서 차단된다
- **SC-006**: 사용자의 90%가 원하는 질문의 답변을 첫 시도에 찾을 수 있다

## Assumptions

- FAQ 항목 수는 일반적으로 50개 미만이므로 페이지네이션이 불필요하다
- FAQ 콘텐츠는 단순 텍스트로 충분하며 리치 텍스트 에디터는 불필요하다
- 모든 FAQ 항목은 공개 상태이며 비공개/초안 기능은 불필요하다
- FAQ 페이지는 조직(Organization) 범위가 아닌 시스템 전역으로 운영되며, 조직 사이드바 "일반" 그룹의 `/org/[slug]/faq` 경로로 접근한다
- 검색/필터 기능은 초기 버전에서는 포함하지 않는다
